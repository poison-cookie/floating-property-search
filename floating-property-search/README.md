# Floating Property Search

サイトごとに登録した検索フォームへ、画面下のフローティング入力欄からすばやく検索語句を送信する Tampermonkey / Violentmonkey 用ユーザースクリプトです。

## 前提

- `dist/floating-property-search.user.js` をユーザースクリプトとして登録します。
- 候補表示はこのスクリプトでは実装せず、既存の `mansion-autocomplete.user.js` をそのまま利用します。
- 候補を使う場合は、`mansion-autocomplete.user.js` も同じブラウザに登録してください。

## 主な機能

- サイトごとに検索ページURL、対象input、検索実行方法を登録できます。
- 対象サイトでは画面左下にフローティング検索フォームを表示します。
- フローティングフォームから検索語句を送信すると、登録済みの検索ページへ移動して対象inputへ値を反映します。
- `Ctrl + /` で、フォーカスモードでは対象フォームへ移動し、フローティングモードではフローティングフォームへフォーカスします。
- 複数タブで同時に検索しても、pending検索語句はタブごとに分離されます。
- 他タブで設定を変更した場合、開いているタブにも反映されます。

## 使い方

1. 対象サイトを開きます。
2. ユーザースクリプトメニューから `検索フォーカス設定を開く` を実行します。
3. 設定画面上部の `マニュアル` で基本操作と注意点を確認します。
4. サイト名、対象ホスト、検索ページURL、対象input、検索実行方法を登録します。
5. フローティングフォームへ検索語句を入力します。
6. Enter または検索ボタンで、登録済み検索ページへ値を送信します。

## 設定項目

- `表示名`: 設定一覧やプレースホルダーに表示する名前
- `対象ホスト`: この設定を有効にするホスト
- `検索ページURL`: 検索フォームが存在するページ
- `対象input name`: 対象inputの `name`
- `CSSセレクタ`: 対象inputを探すセレクタ
- `検索実行`: `none`、`enter`、`button`
- `検索ボタンCSS`: `button` 実行時にクリックするボタンのセレクタ

## 既知の制限

- 候補表示は `mansion-autocomplete.user.js` に依存します。
- ユーザースクリプトマネージャーの実行順によっては、候補表示が初回フォーカスで出ない場合があります。その場合はページを再読み込みしてください。
- 他タブで設定変更が入ったとき、設定UIで入力中の場合はフォームを再描画せず通知だけ表示します。保存時は最新版へマージします。
- Tampermonkey / Violentmonkey 以外のユーザースクリプト環境では、ストレージ同期の挙動が異なる場合があります。
