# Floating Property Search

管理会社サイトなどで、サイトごとに決まった検索フォームへすばやく移動・入力するための Tampermonkey / Violentmonkey 用ユーザースクリプトです。

## 使い方

1. `dist/floating-property-search.user.js` を Tampermonkey に登録します。
2. 対象サイトを開き、左下の `検索` ボタンから設定を開きます。
3. サイト名、対象ホスト、検索ページURL、対象 input の `name` または CSS セレクタを保存します。
4. 動作モードを選びます。

## 動作モード

### 指定フォームへフォーカス

`Ctrl + /` で設定済みの検索ページへ移動し、対象 input にフォーカスします。

### フローティングフォーム

通常時は非表示です。`Ctrl + /` を押した時だけページ左下に小さな検索フォームを表示し、そのフォームへフォーカスします。

フローティングフォームに入力して Enter または `検索` を押すと、設定済みの検索ページへ移動して対象 input に値をセットし、検索結果まで進みます。検索実行方法はサイトごとに `Enter`、`ボタンクリック` から選べます。未指定の場合も、フローティングフォーム送信時はボタンCSS、Enter/form送信の順で検索実行を試します。

## 自作オートコンプリート連携

`mansion-autocomplete.user.js` が保存している候補を、フローティングフォーム内にも表示します。

- 読み取るキー: `mansionAutocomplete.sitePhrases.v1`、`mansionAutocomplete.history.v1`、`mansionAutocomplete.pinned.v1`
- 候補は現在開いているホスト単位で表示します。
- このスクリプトから候補・履歴・ピン止め情報は書き換えません。
- フローティングフォームを開いた瞬間は、未入力でも候補一覧を表示します。
- 候補は矢印キーで移動、Enter または Tab で確定できます。
- 候補確定だけでは検索実行しません。確定後にもう一度 Enter または `検索` を押して実行します。

## 設定の共有

設定画面からサイト設定を JSON でエクスポート / インポートできます。管理会社ごとの検索ページURL、対象 input、検索実行方法を共有できます。

## 検証メモ

React / Vue などの制御 input を考慮し、対象 input への値セット時は native setter と `input` / `change` イベントを発火します。検索ボタンの自動クリックは、サイト設定で有効にした場合のみ実行します。
